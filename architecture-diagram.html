2. Aseprite Binary Export

protosprite-cli · build command

Shell to Aseprite Binary

The CLI finds the Aseprite executable and invokes it with export flags:

  • -b — batch mode (no GUI)
  • --sheet <png> — output spritesheet image
  • --data <json> — output metadata
  • --split-layers --all-layers — separate layers
  • --list-layers --list-tags — include layer/tag info
  • --merge-duplicates --trim — optimize
  • --filename-format "({layer}) {frame}"

Source: cli.ts → findAsperiteBinary(), child_process.execSync()

Intermediate Output

Written to a temp directory (/tmp/protosprite/):

.json Aseprite JSON-Hash metadata

Frame coordinates, sizes, durations, layer names, tag definitions. One entry per ({layer}) {frame} combo.

.png Packed spritesheet image

All frame/layer cels packed into a single atlas by Aseprite's built-in packer.

3. Aseprite Import & Data Model Construction

protosprite-core · importers/aseprite.ts

importAsepriteSheetExport()

Parses the Aseprite JSON export and constructs the core data model:

  • Frame name parsing: Extracts layer name and frame number from ({layer}) {frame} format
  • Layer tree assembly: Builds nested layer hierarchy with group/parent relationships
  • Animation mapping: Maps Aseprite tags → Animation { name, indexStart, indexEnd }
  • FrameLayer mapping: For each frame/layer combo, records sheetPosition (where in atlas), spritePosition (offset in canvas), size, and zIndex

Core Data Structures (data.ts)

TypeScript classes wrapping protobuf messages:

  • SpriteSheetData — top-level container
  • SpriteData — name, size, frames[], layers[], animations[], pixelSource
  • FrameData — index, duration, layers[]
  • FrameLayerData — layerIndex, size, sheetPosition, spritePosition, zIndex
  • LayerData — name, index, isGroup, parentIndex, opacity
  • AnimationData — name, indexStart, indexEnd

Each class has .fromProto() / .toProto() for bidirectional protobuf conversion.

4. Sprite Sheet Packing & PNG Compression

protosprite-core · transform

packSpriteSheet()

Optimizes the sprite atlas layout using bin-packing:

  • Consolidates duplicate frames/regions
  • Re-packs all cels into a minimal atlas
  • Updates all sheetPosition coordinates to match new layout
  • Produces a single combined PNG with embedded pixel data

PNG Compression (optional, default on)

Lossy compression via pngquant:

  • --compression <level> — max colors (2–256, default 256)
  • --uncompressed — disables compression
  • Typically ~30–50% file size reduction
  • Compresses the embedded PNG data in-place before serialization

Source: util/compressPng.ts

5. Protobuf Serialization → .prs

protosprite-core · @bufbuild/protobuf

ProtoSpriteSheet

  • .toArray() → Uint8Array — binary protobuf serialization
  • .toJsonObject() → JSON representation (optional)
  • ProtoSpriteSheet.fromArray(bytes) — deserialize from binary

The data classes convert to/from protobuf messages via .toProto(), then @bufbuild/protobuf handles binary wire encoding.

.prs Binary File Format

Protobuf-encoded SpriteSheet message containing:

  • sprites[] — each with frames, layers, animations, size
  • pixel_source — either:
    • Embedded: PNG bytes inline in the protobuf
    • External: filename or URL reference
  • Each FrameLayer stores atlas coordinates (sheet_position) and canvas placement (sprite_position)

Compact binary — variable-length integers, no repeated strings, ~30-50% smaller than equivalent JSON

6. Geometry Tracing → .prsg

protosprite-geom · trace/index.ts · optional

Triggered by --output-prsg <file>. Generates collision polygons and physics shapes from sprite pixel data.

Step A: Contour Extraction

traceContours() — Marching squares algorithm on the alpha channel. Produces closed polygon contours around opaque regions.

Per-layer mode: Extracts each layer's cel from the atlas, traces individually, offsets to sprite-local coordinates.

Composite mode: Uses renderSpriteInstance() to flatten all layers, then traces the composite image.

Step B: Simplification

simplifyPolygon(contour, tolerance) — Reduces vertex count while preserving shape. Default tolerance: 0.5.

Step C: Convex Decomposition

decomposeConvex() — Splits non-convex polygons into convex components using poly-decomp. Essential for physics engines.

Step D: De-duplication & Pooling

Two-level deduplication dramatically reduces file size:

  • VertexPoolBuilder — Deduplicates Vec2 vertices across all shapes via hash map ("x,y" key)
  • ShapePoolBuilder — Deduplicates (polygon + decomposition) pairs. Shapes reference vertices by index.

Result: frames reference shapes by index → shapes reference vertices by index. Identical cels share a single pool entry.

.prsg Binary File Format

Protobuf-encoded SpriteGeometry message:

  • entries[] — one per sprite, containing:
    • vertex_pool — de-duplicated Vec2 vertices
    • shape_pool — de-duplicated (polygon, convex decomp) pairs using indexed vertices
    • frames[] — per-frame geometry referencing shape pool by index
  • sprite_source — embedded .prs bytes, or external file/URL reference

Wire format definitions

sprite.proto — .prs format

// Top-level
SpriteSheet {
  sprites: Sprite[]
  pixel_source: Embedded | External
}

Sprite {
  name, size, pixel_source
  frames:     Frame[]
  layers:     Layer[]
  animations: Animation[]
}

Frame {
  index, duration
  layers: FrameLayer[] {
    layer_index
    size { width, height }
    sheet_position { x, y }  // atlas coords
    sprite_position { x, y } // canvas offset
    z_index
  }
}

Layer { name, is_group, parent_index, opacity }
Animation { name, index_start, index_end }

sprite_geometry.proto — .prsg format

// Top-level
SpriteGeometry {
  entries: SpriteGeometryEntry[]
  sprite_source: embedded_prs
               | external_prs_file
               | external_prs_url
}

SpriteGeometryEntry {
  sprite_name
  simplify_tolerance
  vertex_pool:  Vec2[]         // de-duped
  shape_pool:   ShapePoolEntry[] // de-duped
  frames:       FrameGeometry[]
}

ShapePoolEntry {
  polygon: IndexedPolygon
  convex_decomposition: IndexedPolygon[]
}

FrameGeometry {
  frame_index
  layers: FrameLayerGeometry[] // per-layer
  composite: CompositeFrameGeometry
}
// Both reference shape_pool by index

7. Three.js Loading Pipeline

protosprite-three · runtime

ProtoSpriteSheetThreeLoader

Async resource loader with caching:

  • loadAsync(url | ProtoSpriteSheet)
  • Fetches .prs binary → ProtoSpriteSheet.fromArray()
  • Resolves pixel source:
    • Embedded PNG: creates Blob URL from inline bytes
    • External: loads from file path or URL
  • Loads texture via Three.js TextureLoader
  • Applies NearestFilter (pixel-art crisp rendering)
  • URL/object caches prevent duplicate fetches

ProtoSpriteSheetThree

Wraps a loaded sheet with Three.js resources:

  • sheetTexture — Three.js Texture from atlas PNG
  • individualTextures — per-sprite textures (if sprites have their own pixel sources)
  • sheetMaterial — ShaderMaterial with custom vertex/fragment shaders
  • getSprite(name | index) → creates ProtoSpriteThree instances

Creates ProtoSpriteInstance (core) internally, then wraps it with a Three.js Mesh.

8. Per-Frame Rendering

protosprite-three · ProtoSpriteThree

Geometry Construction

Each sprite is a single Mesh with a BufferGeometry containing one quad per visible layer:

  • position — Float32 [x, y, z] × 4 vertices per layer-quad
  • uv — Float32 texture coordinates mapping into the sprite atlas
  • vtxIndex — corner index (0-3) for shader effects
  • vtxOpacity — per-vertex opacity
  • vtxMultColor — RGBA multiply tint
  • vtxFadeColor — RGBA fade/blend
  • vtxOutline — RGBA outline color
  • vtxOutlineThickness — outline width

Index buffer pre-filled: two triangles per quad (0,1,3 + 1,2,3). drawRange limits to visible layers.

updateGeometry() — per frame

Called when animation frame changes:

  1. Get current frame from animation state
  2. Calculate Z-order: layer.index × 0.05 + zIndex × 0.05
  3. For each FrameLayer in frame:
    • Skip if layer hidden or parent group hidden
    • Compute quad position: offset + spritePosition
    • Compute UV: sheetPosition / textureSize
    • Fill position & UV buffer arrays
  4. Update draw range to visible layer count

Dirty flags: positionDirty / extraDirty — only updates GPU buffers when state changes.

Animation State Machine

ProtoSpriteInstanceAnimationState (core):

  • advance(ms) — decrements frame timer by delta × speed, advances frames when timer expires
  • Supports forward/reverse playback, looping
  • Emits FrameSwapped, LoopComplete events
  • ProtoSpriteThree re-emits as animationFrameSwapped, animationTagStarted, animationLooped

Custom Shader Pipeline

Vertex Shader

Transforms quad positions, passes UV coords and per-vertex effect attributes (opacity, mult color, fade color, outline) to fragment shader as varyings.

Fragment Shader

Samples the sprite atlas texture, then applies effects in order: outline detection (neighbor alpha sampling), color multiply blend, color fade blend, opacity. Outputs final RGBA with transparency.

Uniforms

map — atlas Texture
invSheetSize — 1/textureSize for UV calculations

Layer Manipulation API

hideLayers() / showLayers()

Toggle layer visibility (expands groups)

setLayerOpacity()

Per-layer alpha override

multiplyLayers(color, opacity)

Color tint via multiply blend

fadeLayers(color, opacity)

Color overlay via fade blend

Edge outline via neighbor sampling

9. protosprite-three-demo

React + Three.js application

App.tsx — Initialization

  1. Creates Scene and ProtoSpriteSheetThreeLoader
  2. Loads bird.prs via loader.loadAsync() → ProtoSpriteSheetThree
  3. Fetches bird.prsg → ProtoSpriteGeometry.fromArray() for geometry overlay data
  4. Calls sheet.getSprite() to create ProtoSpriteThree instances
  5. Bin-packs multiple sprites into the viewport
  6. Adds sprite meshes to the Three.js scene

Renderer.tsx — Render Loop

  • Sets up WebGLRenderer with OrthographicCamera
  • requestAnimationFrame loop calculates delta time
  • Each tick: sprite.advance(delta) updates animation
  • Syncs GeometryOverlay positions if active
  • renderer.render(scene, camera) draws to canvas

Interactive Tabs

  • Sprite Playground: animation select, playback speed, opacity, outline, multiply/fade colors, layer visibility, geometry overlay, sprite count
  • Import: drag-drop Aseprite JSON+PNG export → in-browser conversion to .prs via importAsepriteSheetExport() + packSpriteSheet()
  • Debug: frame stepping, per-frame inspection

GeometryOverlay.ts — .prsg visualization

Renders traced polygons as colored wireframe lines overlaid on the sprite:

■ Green— per-layer polygons ■ Red— per-layer convex decomposition ■ Cyan— composite polygons ■ Yellow— composite convex decomposition
